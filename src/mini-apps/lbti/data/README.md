# LBTI 데이터

## `lbti-framework.json`

LBTI의 지표, 16개 유형, 문항 작성 원칙, 결과 구성, 안전 문구와 기획 출처를 모은 콘텐츠 원본입니다.

- `schema_version`: 파일 구조가 바뀔 때 올립니다.
- `axes`: P–T, W–C, R–O, S–D 네 지표와 양극 설명입니다.
- `source_coverage_matrix`: 교수저·강사저·암기장 논쟁을 기준원과 범위의 두 축으로 나눈 조합입니다.
- `types`: 네 축의 조합으로 생성되는 16개 결과 유형입니다.
- `question_design`: 상황형 이지선다 문항의 원칙과 축별 예시입니다.
- `result_sections`: 결과 화면에 필요한 콘텐츠 블록입니다.
- `safety`: 비진단성 고지와 밈 사용 원칙입니다.
- `sources`: 이 콘텐츠를 구상할 때 참고한 커뮤니티 글과 기사입니다.

## 편집 규칙

- `axes[].id`, 각 pole의 `code`, `types[].code`는 구현과 결과 URL/공유 카드가 참조할 수 있는 안정적인 식별자이므로 출시 후 임의로 바꾸지 않습니다.
- 16개 유형 코드는 반드시 `source → coverage → recall → pace` 순서로 구성합니다.
- 유형명이나 설명을 고칠 때 특정 공부법을 우월·열등하게 만들지 않습니다.
- `PWOS`의 유형명을 유지하는 동안 `required_disclaimer`와 `safety.pwos_required_disclaimer`를 결과 화면에 반드시 표시합니다.
- 실제 문항은 별도 명세가 확정된 뒤 독립 JSON으로 추가합니다. 예시 문항을 곧바로 운영 문항으로 간주하지 않습니다.
- JSON에는 함수, Tailwind 클래스, 화면 상태를 넣지 않습니다. 콘텐츠와 제품 설정만 보관합니다.

## 구현 전 검증 항목

- 네 축이 각각 정확히 2개 pole을 가지는지 확인
- 유형 코드 16개가 중복 없이 모든 조합을 포함하는지 확인
- 모든 유형 코드가 네 축의 정의된 문자만 사용하는지 확인
- 축별 운영 문항이 5개이고 A/B 방향이 한쪽 위치에 치우치지 않는지 확인
- 안전 문구와 출처 링크가 누락되지 않았는지 확인
